# 📧 Email → Telegram (Cloudflare Worker)

Серверлес-пересылка входящих писем с вашего домена прямо в Telegram‑чат.
Работает полностью на инфраструктуре Cloudflare — **ничего не нужно хостить,
поддерживать или держать включённым**.

## 🔄 Как это работает

Входящее письмо → Cloudflare MX → Email Routing → Cloudflare Worker → Telegram Bot API → Ваш чат

## 🤖 Что умеет скрипт

Worker‑скрипт выполняет всю обработку письма на лету, без сохранения данных.

### 📨 Приём письма
- Принимает письмо через Cloudflare Email Routing (поддерживает как обычный текст, так и HTML, а также multipart/alternative письма с вложениями — вложения игнорируются).
- Автоматически декодирует тело письма из Base64 или quoted‑printable в зависимости от заголовка `Content-Transfer-Encoding`.
- Определяет кодировку (charset) из заголовка `Content-Type`. Если кодировка не указана, пробует UTF‑8, затем windows‑1251, и в крайнем случае latin1 — это гарантирует корректное отображение русских и других национальных символов.

### ✂️ Очистка и форматирование
- Извлекает **только тело письма** (текст или HTML). HTML‑теги удаляются, оставляется чистый текст.
- Удаляет все управляющие и непечатные символы (кроме пробелов и переводов строк), заменяя множественные пробелы на один.
- Обрезает итоговый текст до **250 символов** — чтобы не засорять чат.
- Если тело письма не удалось извлечь (например, пустое или нечитаемое), отправляется `(пустое тело)`.

### 🚨 Важность
- Проверяет заголовки `X-Priority`, `Importance`, `Priority`. Если письмо помечено как важное (`high`, `1`, `2`), в начало сообщения добавляется эмодзи ❗.

### 👤 Отправитель
- Определяет email отправителя из заголовка `From`. Если адрес записан в формате `"Имя <email>"`, извлекается чистая почта.
- Первая строка сообщения в Telegram — **адрес отправителя, выделенный полужирным шрифтом**. Затем идёт тело письма.

### 🔒 Фильтр отправителей (опционально)
- Если задана переменная окружения `ALLOWED_SENDERS`:
  - Если значение равно `*`, пропускаются все письма.
  - Иначе значение считается списком email через запятую, и письма доставляются только от адресов из этого списка. Отправители вне списка игнорируются (письмо не приходит в Telegram).
- Если переменная не задана или пуста — фильтр не применяется, все письма пересылаются.

### 📬 Отправка в Telegram
- Сообщение отправляется через Bot API методом `sendMessage` с параметром `parse_mode=HTML`, что позволяет отображать жирный шрифт.
- В случае ошибки чтения письма отправляет сообщение `Ошибка чтения письма`.

## ⚙️ Переменные окружения

Переменные задаются в разделе **Workers & Pages → ваш-worker → Settings → Variables**.

| Переменная | Обязательность | Описание |
|------------|:---:|---|
| `TELEGRAM_TOKEN` | ✅ | Токен Telegram‑бота, полученный у [@BotFather](https://t.me/BotFather) |
| `TELEGRAM_ID` | ✅ | ID чата (личный, группа или канал), куда бот будет слать сообщения |
| `ALLOWED_SENDERS` | ❌ | Разрешённые отправители через запятую. Если `*` или не задано — пропускаются все письма |

## 🚀 Настройка за 5 шагов

### 1. Подготовьте Telegram
- Создайте бота через [@BotFather](https://t.me/BotFather) и сохраните **токен**.
- Получите свой `chat_id` (например, через [@userinfobot](https://t.me/userinfobot)).

### 2. Домен на Cloudflare
DNS домена должен управляться Cloudflare. Если домен ещё не добавлен:
- Зарегистрируйтесь на [dash.cloudflare.com](https://dash.cloudflare.com) и добавьте домен.
- Смените NS‑серверы у регистратора на выданные Cloudflare.

### 3. Создайте Cloudflare Worker
- Перейдите в **Workers & Pages** → **Create application** → **Create Worker**.
- Вставьте код из файла [`worker.js`](./worker.js) (полный исходник в этом репозитории).
- Сохраните и задеплойте.

### 4. Добавьте переменные окружения
- В настройках Worker'а откройте **Variables** и добавьте `TELEGRAM_TOKEN`, `TELEGRAM_ID` и, если нужно, `ALLOWED_SENDERS`.

### 5. Включите Email Routing и направьте письма на Worker
- В панели Cloudflare откройте свой домен → **Email** → **Email Routing**.
- Если маршрутизация ещё не активирована, нажмите **Enable Email Routing**.
- Перейдите в **Routing Rules** и настройте **Catch‑All** (или конкретные адреса) → действие **Send to a Worker** → выберите созданный Worker.

Всё готово! Отправьте тестовое письмо на любой адрес вашего домена — оно появится в Telegram.
